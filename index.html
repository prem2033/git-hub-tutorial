GitHub tutorial for Beginners

GitHub, Inc. is an American multinational corporation that provides hosting for software development and version control using Git. It offers the distributed version control and source code management (SCM) functionality of Git, plus its own features. It provides access control and several collaboration features such as bug tracking, feature requests, task management, continuous integration and wikis for every project.Headquartered in California, it has been a subsidiary of Microsoft since 2018.

GitHub offers its basic services free of charge. Its more advanced professional and enterprise services are commercial.Free GitHub accounts are commonly used to host open-source projects.As of January 2019, GitHub offers unlimited private repositories to all plans, including free accounts, but allowed only up to three collaborators per repository for free.Starting from April 15, 2020, the free plan allows unlimited collaborators, but restricts private repositories to 2,000 actions minutes per month.As of January 2020, GitHub reports having over 40 million users and more than 190 million repositories (including at least 28 million public repositories),making it the largest host of source code in the world.

This blog will walk you through create a local to pusing that repository to remote using git bash.Let's get statted.We need git bash install in our system.Make sure you have git bash installed.if not,to download git bash click here. After downloading install in your system by double clikcking of the exe file.

Step 1: Initilize the git repository to local

To make the repository as a git repository in your local we need to add .git file into your respective repositories.To add a git repository,first we need to land tp respective repository,then right click on empty space and click git bash here,type below command to add .git file in your respective repository.you will get the message Reinitialized existing Git repository in E:/git-tut/.git/,In my case folder was residing in E/git-tut.

//Initialized empty Git repository
$ git init
Step 2: Stage and commit the file

Now we are ready to stage and commit the code to this git tree,by default the tree will be resides on the master branch.we can navigate through as many branch you want by creating new branch to master.Git follw three layer architecture to add the code to branch.To push the file to existing branch first we need to stage the files then commit the files to that branch.The three layer Architecture diagram are shown in below picture.commnd to add the file to staging and commit is shown below.

//add file to staging
$ git add .
//add file to commit
$ git commit -m "commit message"
Step 3: Add remote reposetry

We are successfully staged and commit the file to current branch.Now it time to push to remote git repository.At this satge we need a github account.if you dont have an account,create one.Now click on plus sign on github account to create a new repository.Write a meaningful name,choose type of repository you want to create,add read me file(optional,we can add later as well) and click create repository.

Once the repository will get created github will land on below page.Now we are ready to add github remote repository to local repository.To add the remote repository,click on code and copy https link to the git repository.Type below command to add remote repository to your local.

//to add remote repository
$ git remote add origin https://github.com/prem2033/git-tut.git
Step 4: Push the code to remote repo

we are set with the remote repository to our local,Now its time to push the files to remote repository from the local repository.To push the file type the below command.After successfully pushing the files we can check the respective remote folder.

//push the code
$ git push origin branch_name
